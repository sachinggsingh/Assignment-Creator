================================================================================
POSTMAN DEMO — Assignment API (JWT auth + Gemini)
Base URL: http://localhost:3000
Start dev server first: npm run dev
================================================================================

STEP 1 — LOGIN (get JWT)
------------------------
Method:  POST
URL:     http://localhost:3000/api/auth/login

Headers:
  Content-Type: application/json

Body → raw → JSON:

{
  "email": "teacher@example.com",
  "password": "yourpassword"
}

Expected: 200 OK
Response:
{
  "user": { "id": "...", "email": "...", "name": "...", "role": "teacher" },
  "token": "eyJhbGciOiJIUzI1NiIs..."
}

Copy the "token" value for the Authorization header below.

(Register first if needed: POST /api/auth/register with name, email, password)


STEP 2 — SET ROLE (if role is null after register)
--------------------------------------------------
Method:  PATCH
URL:     http://localhost:3000/api/auth/role

Headers:
  Content-Type: application/json
  Authorization: Bearer PASTE_TOKEN_HERE

Body:
{
  "role": "teacher"
}


================================================================================
1) CREATE ASSIGNMENT (Gemini generates the paper)
================================================================================

Method:  POST
URL:     http://localhost:3000/api/assignments

Headers:
  Content-Type: application/json
  Authorization: Bearer PASTE_TOKEN_HERE

Body → raw → JSON:

{
  "dueDate": "2026-06-15T23:59:59.000Z",
  "questionTypes": [
    {
      "type": "MCQ",
      "numberOfQuestions": 3,
      "marksPerQuestion": 2
    },
    {
      "type": "Short Answer",
      "numberOfQuestions": 2,
      "marksPerQuestion": 5
    },
    {
      "type": "True/False",
      "numberOfQuestions": 2,
      "marksPerQuestion": 1
    }
  ],
  "additionalInstructions": "Focus on data structures and algorithms. Keep language suitable for undergraduate students."
}

Valid question type values:
  MCQ | Short Answer | Long Answer | True/False | Coding | Essay

Expected success: 201 Created


================================================================================
2) LIST YOUR ASSIGNMENTS
================================================================================

Method:  GET
URL:     http://localhost:3000/api/assignments

Headers:
  Authorization: Bearer PASTE_TOKEN_HERE


================================================================================
3) GET ONE ASSIGNMENT BY ID
================================================================================

Method:  GET
URL:     http://localhost:3000/api/assignments/REPLACE_WITH_ASSIGNMENT_ID

Headers:
  Authorization: Bearer PASTE_TOKEN_HERE


================================================================================
AUTH ENDPOINTS REFERENCE
================================================================================

POST   /api/auth/register   { name, email, password }
POST   /api/auth/login      { email, password }
POST   /api/auth/logout      (clears cookie; browser only)
GET    /api/auth/me          (Bearer or auth_token cookie)
PATCH  /api/auth/role        { role: "teacher" | "student" }


================================================================================
COMMON ERRORS
================================================================================

401 Unauthorized     → missing/invalid JWT
403 Forbidden        → user is not a teacher (POST assignments only)
400 Bad Request      → invalid JSON body
429 Too Many Requests→ Gemini rate limit
503 Service Unavailable → missing/invalid GEMINI_API_KEY

Required .env variables:
  JWT_SECRET=your-long-random-secret
  GEMINI_API_KEY=your-google-ai-key
  GEMINI_MODEL=gemini-2.0-flash
  MONGODB_URI=your-mongodb-connection-string
